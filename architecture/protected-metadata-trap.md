# Metadata Trap Protector

**Source:** `src/protected/metadata_trap.rs` (~2043 lines)

The largest module. Injects metadata into image files for the `Light` protection level. Operates at the byte level ŌĆö the `DynamicImage` API cannot preserve injected text chunks through encode/decode cycles. Estimated latency: 2ms.

## Key Behavior

- **`apply()` returns `Cow::Borrowed(img)` unchanged** ŌĆö metadata injection cannot survive through the `DynamicImage` API
- **`apply_bytes()` / `inject_bytes()`** ŌĆö The actual metadata injection happens here
- **Pipeline routes `Light` level through `apply_light_bytes()`** which encodes, injects metadata, then decodes

## Metadata Types

### IPTC Data Mining Inhibitor (DMI)

Seven variants mapped to IPTC XMP property names:

| DmiValue | IPTC Property |
|----------|---------------|
| `Unspecified` | Not injected |
| `Allowed` | `Iptc4xmpExt:DMI-Allowed` |
| `Prohibited` | `Iptc4xmpExt:DMI-Prohibited` |
| `ProhibitedAiMlTraining` | `Iptc4xmpExt:DMI-Prohibited` |
| `ProhibitedGenAiMlTraining` | `Iptc4xmpExt:DMI-Prohibited` |
| `ProhibitedExceptSearchEngineIndexing` | `Iptc4xmpExt:DMI-Prohibited` |
| `ProhibitedSeeConstraints` | `Iptc4xmpExt:DMI-Prohibited` |

### DMI Auto-Mapping

When no explicit DMI value is set, the protector auto-maps from `ProtectionLevel`:

- Light ŌåÆ `Prohibited`
- Standard ŌåÆ `ProhibitedAiMlTraining`

### Legal Metadata

When `inject_legal_claims` is enabled, injects:
- Copyright holder, contact email, license URL
- Usage terms, creation date, AI training constraints
- Web statement of rights

## Format-Specific Injection

### JPEG

Injects four marker types:
1. **EXIF (APP1)** ŌĆö Basic EXIF orientation
2. **IPTC-IIM (APP13)** ŌĆö Photoshop 3.0 identifier + IPTC dataset records
3. **XMP (APP1)** ŌĆö Full XMP packet with IPTC extension properties
4. **COM** ŌĆö Human-readable key-value pairs (Copyright, Creator, Contact, UsageTerms, AIConstraints) + structured binary COM (`cloakrs:v1:`)

**External visibility**: Legal fields are stored as separate COM markers. External tools like `exiftool` require the `-a` flag to read all COM markers (not just the first). XMP DMI/TDM fields are visible via standard XMP extraction.

### PNG

Injects two chunk types:
1. **tEXt** ŌĆö Key-value text chunks (Copyright, Creator, Contact, UsageTerms, AIConstraints, X-Protection-Seed, Description, etc.)
2. **iTXt** ŌĆö International text chunks (for XMP data, UTF-8 encoded)

**External visibility**: tEXt chunks map to standard exiftool fields (e.g., `exiftool -Copyright`). XMP in iTXt is extractable via standard XMP tools.

### WebP

Injects:
1. **META chunk** ŌĆö Contains XMP data generated by `generate_xmp_notice()`: DMI, TDM, seed, plus legal fields via standard XMP namespaces (`dc`, `xmpRights`, `photoshop`, `stegoeggo`)
2. **EXIF chunk** ŌĆö UserComment with seed
3. Updates RIFF file size header

**External visibility**: XMP DMI/TDM fields are visible via `exiftool`. Legal fields (Copyright, Creator, Contact, UsageTerms, AIConstraints) are now embedded as standard XMP properties (`dc:rights`, `dc:creator`, `photoshop:Credit`, `xmpRights:UsageTerms`, `stegoeggo:AIConstraints`) ŌĆö external parser support for these namespaces determines visibility.

## Seed Extraction

```rust
pub fn extract_seed_from_image(img_bytes: &[u8]) -> Option<u64>
```

Extracts the `X-Protection-Seed` value from:
- PNG tEXt chunks
- JPEG COM markers
- WebP META chunks

Used by the verification pipeline to recover the seed for stego extraction.

## XMP Generation

Generates IPTC-compliant XMP packets. The `<rdf:Description>` opening tag is closed
before any legal child elements are inserted, and `dc:rights` / `xmpRights:UsageTerms`
are wrapped in `<rdf:Alt><rdf:li xml:lang="x-default">ŌĆ”</rdf:li></rdf:Alt>` containers
so external RDF parsers (e.g. `exiftool`) can read the legal fields:

```xml
<x:xmpmeta xmlns:Iptc4xmpExt="..." xmlns:dc="..." xmlns:xmpRights="..." xmlns:photoshop="...">
  <rdf:RDF>
    <rdf:Description rdf:about=""
      Iptc4xmpExt:DMI-Prohibited="ProhibitedAiMlTraining"
      tdm:reserve_tdm="1"
      stegoeggo:ProtectionSeed="...">
      <dc:creator>
        <rdf:Seq>
          <rdf:li>Test Author</rdf:li>
        </rdf:Seq>
      </dc:creator>
      <dc:rights>
        <rdf:Alt>
          <rdf:li xml:lang="x-default">Copyright (c) Test Corp</rdf:li>
        </rdf:Alt>
      </dc:rights>
      <xmpRights:UsageTerms>
        <rdf:Alt>
          <rdf:li xml:lang="x-default">All rights reserved</rdf:li>
        </rdf:Alt>
      </xmpRights:UsageTerms>
      <xmpRights:WebStatement>https://example.com/rights</xmpRights:WebStatement>
      <photoshop:Credit>legal@test.com</photoshop:Credit>
      <stegoeggo:AIConstraints>No AI training</stegoeggo:AIConstraints>
    </rdf:Description>
  </rdf:RDF>
</x:xmpmeta>
```
