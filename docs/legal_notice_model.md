# Legal Notice and Evidence Model

## Purpose

stegoeggo helps content owners embed rights-reservation and AI-training restriction metadata into images at distribution time. The goal is to make copyright and AI-training restrictions discoverable by metadata-aware systems, and to provide redundant evidence channels that the restrictions were present when the image was distributed.

## Legal/Evidence Theory

stegoeggo supports a notice-and-evidence workflow:

1. **Notice**: Embedding visible, standards-compliant metadata (IPTC DMI, XMP, EXIF, COM markers) that communicates AI-training restrictions to any system that reads image headers.
2. **Evidence**: Embedding redundant steganographic markers that can demonstrate the restrictions were present at the point of distribution, even if header metadata is stripped.
3. **Authenticated provenance** (optional): Using HMAC-SHA256 to cryptographically sign steganographic payloads, proving they were generated by a party with the configured secret.

This tool helps demonstrate that a rights reservation was present and discoverable at distribution time. It does not prove infringement, does not prove a model was trained on a specific image, and does not determine legal outcomes.

## Metadata Channels

stegoeggo embeds metadata through multiple channels:

| Channel | Standard | Survives | Notes |
|---------|----------|----------|-------|
| IPTC DMI tags | IPTC Photo Metadata Standard | Header-aware tools | Primary AI-training restriction signal |
| XMP packets | IPTC/XMP | Header-aware tools | Structured metadata with DMI values |
| EXIF comments | EXIF | Most tools | Additional metadata carrier |
| COM markers | JPEG COM | JPEG-aware tools | Text metadata in JPEG files |
| tEXt/iTXt chunks | PNG spec | PNG-aware tools | Text metadata in PNG files |
| XML chunks | WebP spec | WebP-aware tools | Text metadata in WebP files |

## Optional Steganographic Markers

When enabled, steganographic markers provide a redundant evidence channel:

- **LSB embedding** (PNG/WebP): Payload embedded in least-significant bits of pixel channels, with spread-spectrum and error-correcting codes for robustness.
- **DCT embedding** (JPEG): F5-style perturbation of DCT coefficients, with seed storage in quantization tables.
- **Tiled embedding**: Crop-resistant mode that embeds the full payload in each tile of the image.

Steganographic markers are best-effort. They survive some transformations (file copy, re-hosting, format-preserving re-encoding through stegoeggo) but not others (generic re-encoding, resize, crop without tiling, metadata stripping tools).

## Optional Authenticated Provenance (MAC Key)

When a MAC key is configured, steganographic payloads are signed with HMAC-SHA256. This allows verification that:

- The payload was generated by a party with the configured secret
- The payload has not been modified since embedding

Without a MAC key, payloads use a non-cryptographic CRC32 checksum. This is suitable for development and testing but does not provide cryptographic authentication.

## Common Transformations and Their Effects

| Transformation | Metadata | Stego |
|---------------|----------|-------|
| File copy / re-hosting | Survives | Survives |
| Social media upload | Usually stripped | Usually destroyed |
| Screenshot | Lost | Lost |
| Generic JPEG re-encode | Stripped | Destroyed (unless using stegoeggo fast path) |
| Resize | Resampled | Destroyed |
| Crop | Clipped | Destroyed (unless tiled) |
| Metadata stripping tools | Stripped | Survives (still extractable) |

## Operational Recommendations

1. **Use metadata as the primary channel**: IPTC DMI and XMP are standards-compliant and detectable by any metadata-aware system.
2. **Enable steganographic markers for redundancy**: They provide evidence that the restrictions were present even if headers are stripped.
3. **Use a MAC key for authenticated provenance**: When you need to prove the payload was generated by an authorized party.
4. **Serve protected images directly**: Downstream consumers may strip metadata or re-encode images. The strongest evidence is the image as you served it.
5. **Consider tiled embedding for crop resistance**: Use `with_tile_size()` to embed payloads in each tile.
